Webの知識を見つけられる Web developer library

10代向けのWeb開発を学ぶWebサイト

Role

UXデザイナー, リサーチ

Project duration

2022.9 to 2022.10


Project Goal

初めてWeb開発をする10代に役立つWeb開発情報を提供します。

Target Audience

10代のプログラミング未経験者


Key challenges

Web開発が初めてとなる10代を助けるツールをデザインします。

Techniques
Figma, Word, Excel

Slideshow

Google Slides

OVERVIEW

Google UX Design Certification's assignment/課題

Web開発に必要な情報に素早くアクセスできるWebサイトをつくります。

  • インタビューを行い、どのようにWeb開発の勉強をしたのか聞いていきます。
  • プロトタイプの制作と改善
  • ユーザーテスト

ユーザーリサーチについて

インターネットからWeb開発の情報を見つけることが手間:

検索すると大量の情報が出てきて、どこから手をつけて良いのかわからない。


何から始めたら良いのかわからない:

大学からWeb開発の勉強を始めたので、何から始めたら良いのかわからない。


インターネットからWeb開発の情報を見つける:

周りに詳しい人がいないので自分でその技術について調べた。


インタビュー対象者:

男性4名、女性1名(日本、オランダ、中国)


ペルソナ:

ペルソナとして自分の作品集を作りたい学生リンダと文化祭のウェブサイトを作りたい学生ヒューゴを構築しました。

初期デザインのコンセプト:

Web開発に役立つWebサイトを集めてWeb開発が初めての10代にその情報を提供します。

ユーザーテストについて

手順:

1. ツールというページを見つけてもらう

2. ツールの一覧からWeb開発ソフトのVisual Studioを見つけてもらう。

3. Web開発ソフトのVisual Studioの詳細ページに移動してもらう。

テスト協力者:

男性5名(日本、オランダ、中国)


調査結果:

1.専門用語は使わない。

2.スマートフォンのものは文字を大きくする

3.説明は簡潔にまとめる。

モックアップについて

改善:

IDE(統合開発環境)からツールにタイトルを変更しました。文字サイズを大きくと説明は簡潔に変更しました。

モックアップの説明:

1.Web開発について学べるオンラインサイトを紹介する、学習サイトのページです。

2.プロが作ったWebサイトを紹介するインスピレーションのページです。

3.実際にプログラミングをするときに必要なツールを紹介するページです。

結論と学び

結論:

5名のユーザーテストの結果、情報(Web開発ツール)を見つけるのに約1分程度でできることがわかりました。

学び:

初期Webサイトには情報を多くいれましたが、それが逆に利用者にとってストレスになることがわかりました。適切な情報量を考えることが大切だと学びました。
